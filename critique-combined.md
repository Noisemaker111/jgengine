# JGengine combined critique — actionable list

Merged from `critique-gpt.md` (security/persistence/lifecycle audit), `critique-grok.md` (product/API-surface audit), `critique-sonnet.md` (repo-mechanics audit). Independent passes; overlaps confirm signal. Source tags: **[G]** gpt, **[K]** grok, **[S]** sonnet. Dedup'd, one flat priority-ordered list. Delete the three source files once this is triaged into issues.

Severity: **P0** exploitable/ship-blocker/product-lie · **P1** structural debt/data-loss/broken-authority · **P2** real pain · **P3** polish.

---

## P0 — do first

1. **Public GitHub proxy is a token-backed confused deputy.** [G] `packages/github/src/server.ts:178-202`, `apps/web/src/routes/api/github-proxy.ts:5-13` — arbitrary REST/GraphQL through the server's token, CORS `*`, publicly cacheable. Kill the general proxy; replace with named allowlisted operations.
2. **Private multiplayer sessions aren't private.** [G] `packages/ws/src/readsHandler.ts:64-69`, `host.ts:280-318`, `packages/convex/src/server.ts:838-869,526-531` — private IDs appear in listings and a direct `serverId` bypasses the join-code check everywhere (ws, HTTP, Convex).
3. **Convex identity is caller-supplied.** [G] `packages/convex/src/hostedServer.ts:178-184`, `server.ts:175-194`, shipped in `examples/convex-host/*` — anonymous mode accepts any claimed `externalId`; trivial player impersonation.
4. **Shared world snapshot leaks every player's private state to every client.** [G] `packages/core/src/runtime/gameContext.ts:1463-1471`, `worldChannel.ts:83-87`, `packages/ws/src/worldHost.ts:93-103` — one global snapshot broadcast, no per-viewer projection. Related: **[S]** broadcast fan-out is O(all connections) not O(room) (`hostRouter.ts:148-204`), compounding the exposure and the DoS surface.
5. **Multiplayer defaults are silently not multiplayer.** [K] `authority` defaults to client/presence-only in `packages/core/src/runtime/adapter.ts`; only claudecraft uses `authority: "server"`. Agents wire `ws()`/`convex()` and ship independent sims believing it's shared. Rename to make presence-only explicit (`wsPresence()`), gate intake on `authority: "server"` + host recipe.
6. **No rate limiting outside chat.** [S] `packages/ws/src/hostRouter.ts:158-160,342-371` — pose/runCommand/join/browse/voice unthrottled; combined with #4's global broadcast this is a real DoS ceiling, not defense-in-depth.
7. **Reconnect can duplicate committed mutations.** [G] `packages/ws/src/createWsBackend.ts:169-182,315-323` — no stable op ID, no host-side dedup ledger. Lost reply after a purchase/attack/reward commits it twice on replay.
8. **Global mutation queue serializes every room on one host.** [G]+[S] agree: `packages/ws/src/host.ts:138-143` — one promise chain gates join/run/tick/flush for *every* room; one slow write for room A stalls all unrelated rooms. Direct contradiction of "scale is a default."
9. **Client input reaches game logic unvalidated.** [G]+[S] agree: `packages/core/src/commands/commandRegistry.ts:15-46`, `packages/node/src/worldHost.ts:80` — TS types are cast onto raw network `unknown`/attacker input with no runtime parser/codec at the boundary.
10. **Hosted-world sessions have no persistence by default; no clean shutdown.** [S] `packages/node/src/worldServer.ts:50` → in-memory store, no plug point; **[G]** confirms no atomic flush (`host.ts:179-202`, sequential awaits, no rollback); **[S]** no SIGINT/SIGTERM hook anywhere in node/ws calls `host.stop()`. A redeploy or OOM-kill loses everything, not just a flush window.
11. **`gameContext.ts` is a 1,700-line god factory — all three audits independently flag this as the top architectural problem.** [G]§5.1 [K]§G1.1 [S]§Phase1 `packages/core/src/runtime/gameContext.ts`. Every opt-in feature is a hand-wired `features.x ? create : undefined` branch — the exact "engine file edited for every new variant" smell CLAUDE.md names. Root cause of the save/replication gaps in #14 and the editor's parallel `applyMutating` (467 lines, 43 cases, `packages/core/src/editor/commands.ts:250-717`). Fix: typed feature descriptors that own creation/snapshot/replication/dispose; context composes them, doesn't know them. **Status: half-done, not closeable.** The descriptor *seam* landed (`#849`) — no more hand-wired branches, `save`/`replicate` wiring collapsed. The *thinning* never happened: `gameContext.ts` is still ~2000 lines, descriptor bodies still inline, and `time`/`pose`/`possession`/`motion` still tick via hand calls in the runners (`headlessRunner.ts`, `hostedGameRunner.ts`), not through a descriptor `tick()` — `FeatureBuild` has no `tick`/`dispose` field at all. Full fix tracked in open issue **#842** (systems own tick/save/replicate/dispose, per-system scheduling). Do not mark this item done until #842 closes.
12. **"Never Kenney" is already violated in a shipped game, silently broken.** [S] `Games/wreckway/src/game/assets.ts:7` and `models.ts` reference ~20 `kenney-*` ids that don't exist in `packages/assets/src/sources/index.ts` — every reference no-ops to a fallback shape. Simultaneously a hard-rule violation and a live rendering bug; no CI catches it (`packages/assets/src/verify.ts:21-61` never checks provider).
13. **PR CI doesn't compile or test code.** [G]+[S] agree: `.github/workflows/ci.yml:13-49` — only the ~30s `quick` job runs on `pull_request`; build/types/tests/web-build/smoke are `push`-only and skipped-job-passes-gate. A PR can merge green without compiling. Canonical type gate is *currently* red on main from a stale `.claude/skills/jgengine-ui/api.md` [G].
14. **"Whole-world save" doesn't save the whole world.** [G] `gameContext.ts:1449-1548` omits time, stats, cosmetics, card piles, turn loops, motion/pose/possession state from snapshot/save; `turnLoop.ts:43-63` has an unregistered `restore`. Save/load silently corrupts games built from advertised primitives.
15. **Tauri desktop disables CSP and loads arbitrary URLs into that unrestricted webview.** [S] `apps/desktop/src-tauri/tauri.conf.json:21` (`"csp": null`), `packages/jgengine/src/desktop.ts:141-153,427-430` (`--url` loads any HTTPS URL).
16. **Product claim vs. reality mismatch (own it or fix it).** [K] README/keywords say "engine"; culture ships genre-universal kits. Pick one sentence ("agent-first game framework SDK") and demote/delete/promote everything else. Same bucket: AGPL under-communicated as a commercial kill-switch — ship a licensing FAQ before 1.0 commercial talk [K].

## P1 — before adding more surface

17. **Editor RPC/CLI (the one genuinely untrusted-input path) has the weakest type safety.** [S] `packages/editor/src/mcp/cli.ts:68`, `stdioServer.ts:147` cast unvalidated game-authored modules with `as never`; `loadGameLayers.ts:2-11` returns `unknown`, zero schema validation before it hits a live session.
18. **Editor documents have no validation/migration boundary.** [G]+[S] agree: `packages/core/src/editor/document.ts:309-317` only checks "is an object"; `EditorDocument.version: 1` has no upgrade path despite nested `migrateTerrainSnapshot` existing (`editor/types.ts:104-116`). Add one authoritative decoder/migrator with path-specific diagnostics; fuzz it.
19. **Editor object IDs aren't globally unique.** [G] add/import/merge dedupe only within their own collection (`editor/commands.ts:336-373`, `document.ts:98-109`) while selection/parenting/removal treat IDs as document-global.
20. **Editor RPC reports success on rejected mutations.** [G] `packages/editor/src/session.ts:378-393` returns `{ok:true}` after a locked/cyclic `set_transform` rejection — an agent can believe an edit landed and ship unchanged content.
21. **Membership/join/leave is not idempotent or reconnect-safe.** [G] `hostedGameRunner.ts:78-83` fires `onNewPlayer` on every re-join of an existing ID; `worldChannel.ts:75-77` closes without leaving; membership keyed by user ID only (one of two tabs can drop the shared player). No session-resume handshake tracks previously-joined servers across reconnect (`hostRouter.ts:554-597`, `createWsBackend.ts:285-323`).
22. **No sequencing/coalescing on remote input.** [G] `packages/shell/src/inputSink.ts:19-25` fires independent, unordered `runCommand` promises per changed frame and swallows failures — a delayed "held" frame can arrive after "released."
23. **No revision/epoch/ack/resync on world diffs.** [G] `packages/core/src/runtime/worldMirror.ts:31-34` blindly applies any diff; stale/duplicate/out-of-order frames can regress client state with no baseline-resync path.
24. **Module deletions aren't represented in world diffs.** [G] `worldReplication.ts:11-99,220` — no `removedModules`; clients keep stale module state forever once a module disappears server-side.
25. **Snapshot boundaries are mutable, not owned.** [G] `hostedWorldSession.ts:28-34`, `entityStore.ts:287-310` return live object references — a "saved" record can mutate after the save with no cadence/revision tracking catching it. Same root issue as **[S]**'s list of mutable-getter leaks (`ObservableKeyedStore.mapSnapshot`, `InputSnapshot`, `connectedPlayers` getters).
26. **Concurrent SQL feed append isn't atomic.** [G] `packages/sql/src/sqlPersistence.ts:284-301` — read-modify-upsert in app memory, no row lock; concurrent writers drop each other's entries. Tested only against `pg-mem`, not real Postgres contention [K confirms: convex/sql/ws all under-tested against real backends].
27. **Convex reads/ticks scale by total data, not touched partitions.** [G] `packages/convex/src/server.ts:257+,871-880`, `hostedServer.ts:413+` — hydrates all profiles/chunks per command, collects every server per scheduled tick. Hits Convex transaction ceilings well before stated scale target. [K] adds: convex has the largest per-adopter orphan ratio (22 unadopted exports) for a second full backend — either make it the recommended hosted story with a gallery game, or mark experimental and shrink it.
28. **Unchanged ticks cost O(total world); replication isn't AOI-scoped.** [G]+[K] agree: `worldReplication.ts:73-130` `JSON.stringify`s every entity/stat/store/module every commit regardless of change, then every client reparses it; no interest-management/spatial subscription exists. Fix: dirty-flag/mutation-journal change detection + viewer-aware AOI before publishing more multiplayer surface.
29. **No resource disposal contract — audio, GPU textures, game lifecycle all leak.** [G]: loop-audio handles don't actually stop/disconnect sources (`audioEngine.ts:133-158`); listener movement doesn't update spatial attenuation (`audioEngine.ts:142-164`); painted canvas textures aren't disposed on entity/scene replacement (`modelRender.ts:84-98`, `GamePlayerShell.tsx:441-598`); `GameLoop` has no `onDispose` seam at all (`defineGame.ts:54-62`). Needs one lifecycle scope/disposer registry, tested under StrictMode + mode-churn.
30. **CLI version line is decoupled from SDK version line.** [G] `packages/jgengine/package.json` CLI at 0.8.5 scaffolds SDK deps at its own version (`create.ts:10`, `templates.ts:127+`) vs. actual SDK 0.10.0 — scaffolds can be stale/unresolvable.
31. **Publish workflow omits packages that should ship.** [G] `.github/workflows/publish.yml:39,51` excludes `@jgengine/editor` and `@jgengine/github` from changelog/publish despite source changes triggering the workflow; hardcoded in `packaging.test.ts:40`.
32. **Content-governance rule ("author in editor, never hardcode geometry") is violated by 7 of 10 games, and nothing gates it.** [S] Only tower-guard and studio-showcase actually use `<AuthoredScene>`. Worst offenders: `Games/the-robots/src/game/world/level.ts:122-372` (hand-rolled mesh gen + ~90 magic-number placements), `Games/vice-isle/src/game/world/districts.ts:12-84`, `Games/loopline/src/game/world/setup.ts:14-73`. `check-game-shape.ts` checks layout only, no content-level check. [K] adds: this is a doctrine-vs-tool gap — editor MVP (place marker/path/scatter/terrain, save, AuthoredScene parity) needs an integration test, and large coordinate literals in `Games/*` should fail lint unless tagged exception.
33. **Wildcard package exports make internal file layout an accidental public API.** [G] broad `"./*"` exports, `@internal` doesn't enforce a boundary, test fixtures (`@jgengine/node/testFixtures`) are importable. Needs a reviewed, generated manifest of supported subpaths + tarball-install tests against clean consumers (no test currently proves a *published tarball* works, only workspace-source aliases) [G confirmed by K's G8.2/cold-start gap].
34. **358 orphaned exports (no game adopter, no skill example) — a design smell, not bookkeeping.** [K] `scripts/api-orphan-baseline.json`, core alone 177. Combined with **[G]**'s 570 `@internal` tags on 3,151 exports (18%) sitting alongside flagship public API mistagged as internal (`ai/spawnPoint.ts:11` re-exported as public at `authoring.ts:49`) — the internal/public/library boundary is currently meaningless. Enforce: new export needs adopter or skill example or explicit `@internal` in the same PR; ratchet the baseline so it can only shrink.
35. **Public APIs leak internal engine mechanics CLAUDE.md explicitly bans.** [G]+[K] agree: `PhysicsWorldConfig` exposes solver/cellSize knobs as the primary surface; `spawnDirector.ts` exposes a raw `biasStrength` exponent and leaks PRNG state (`rng: number`) in return types (`heatSystem.ts:36`, `spawnDirector.ts:39`) — contrast with the correct injected-RNG pattern elsewhere.

## P2 — real pain, schedule after P0/P1

36. **God components beyond gameContext.** [G]+[S] agree: `GamePlayerShell.tsx` (~2,250 lines), `DevtoolsOverlay.tsx` (~1,225), `EditorChrome.tsx` (~1,205). Split by lifecycle/ownership boundary, not line count.
37. **`gameContext`'s three competing mental models for entity/behavior logic.** [K] bag-of-entities (`entityStore.ts`) + subsystem facades + bolted-on `Behaviour` lifecycle, with no ADR saying which owns tick logic.
38. **Shell is a second full engine bolted onto the only rendering path.** [K] cameras/audio/weather/terrain/postfx/HUD/multiplayer sink all in one package with R3F/React as a hard peer dep — no non-React game path exists despite core being renderer-neutral. Extract a headless playable runner (tick+input+snapshot) from the R3F presentation layer.
39. **Skills are the real docs and are too large to be load-bearing.** [K] `jgengine/SKILL.md` 800+ lines; every session pays full intake cost. Cap skill bodies, push encyclopedic content into generated `api.md`/`capabilities.md`, measure success by "first correct import."
40. **Deep-path-only imports with no curated barrels.** [K] `packages/core/src/index.ts` exports nothing but VERSION; every call site needs an exact file path, and skills/disk drift causes agents to invent paths. Add curated per-domain entrypoints generated from the same pipeline as `api.md`.
41. **Screenshot/visual verification tooling is a known hang/flake vector.** [K] documented shoot hangs on heavy WebGL; `summarizeEnvironment` proves content exists, not that it looks good — no cheap automated floor exists between "content present" and "an agent looked at it."
42. **File persistence has a path-traversal-shaped encoding bug and no recovery story.** [G] `packages/node/src/persistence.ts:100-139` — `encodeURIComponent("..")` stays `".."`; in-process-only locks never removed; no fsync/backup/corruption-quarantine policy. Must be labeled single-process dev persistence until fixed.
43. **Unbounded ingress everywhere.** [G] `packages/node/src/wsServer.ts:22-40` no `maxPayload`/origin policy/connection cap/heartbeat; `protocol.ts:36-50` unbounded command/feed/tag sizes; `hostRouter.ts:574-588` unbounded promise queue; asset downloads (`packages/assets/src/download.ts`) fully buffer and `unzipSync` with no size/ratio/file-count caps and only-optional hash verification.
44. **Numeric inputs aren't validated as finite anywhere near the clock.** [G] `packages/core/src/time/simClock.ts:104-172,248-252` accepts `Infinity`/`NaN`, can create immortal timers; 10,000-fire catch-up cap silently drops work with no policy stated.
45. **Autosave clock defaults to a constant.** [G] `hostedWorldSession.ts:43-113` uses `0` whenever `now` is omitted — a positive `saveIntervalMs` can never actually elapse.
46. **Games hardcode duplicate tooling config instead of sharing a base.** [S] all 10 games + 2 non-web apps repeat an identical `devDependencies` block with no shared config; unexplained major-version split (`apps/web` on vite 8, everything else on vite 6/4).
47. **`jgengine create` prints corrupted mojibake output**, and the editor subcommand hard-requires `bun` on PATH with no npm/pnpm fallback despite `--pm` existing elsewhere. [S] `packages/jgengine/src/create.ts:37,128-158`.
48. **Vercel deploy trigger is far broader than the site's actual dependency.** [S] `apps/web/vercel.json:2` diffs against `apps/web apps/dev packages Games .claude/skills` wholesale though the site only depends on `@jgengine/github` — any unrelated Games/* tweak forces a full redeploy.
49. **CREDITS.md and assets README are stale relative to actual asset sourcing.** [S] KayKit/Quaternius power the last 5+ art PRs but aren't in `CREDITS.md`; `packages/assets/README.md` documents a `kenney.ts` source module and `pull kenney-food` example that don't exist — stale docs pointing at the explicitly-banned provider.
50. **Two fully dead subsystems still ship in `dist`.** [S] `packages/core/src/store/engineStore.ts` (`EngineStore`) and `store/features.ts` (`composeStateFeatures`) — zero imports, no tests, superseded already. Delete.
51. **Real circular type-level coupling between `world` and `editor` domains.** [S] `world/scatterRegion.ts`, `world/vegetation.ts` import `../editor/types` while `editor/commands.ts`/`document.ts`/`types.ts` import `../world/terraform`. Compiles fine (type-only) but is genuine bidirectional domain entanglement; same pattern between `combat/death.ts` ↔ `game/events.ts`.
52. **Testing gaps concentrated exactly where risk is highest.** [S]+[G] agree: `runtime/gameRuntime.ts` join/tick/dirty logic has no direct unit test; 0 of 14 editor `.tsx` panels are tested; convex package has no dedicated test file per module; multiplayer race conditions (concurrent writers, out-of-order delivery, mid-session reconnect at the router level) are untested end-to-end. [G]'s required-suite list (exploit paths, lifecycle/resource leaks, real-backend integration) in `critique-gpt.md` Phase 6 is the fullest version of this — use it as the test backlog.
53. **Editor React perf: force-update via 3 `useState` counters instead of `useSyncExternalStore`.** [S] `packages/editor/src/EditorApp.tsx:117-120` re-renders the entire overlay subtree (`PerfProbe`, cameras, gizmo, terrain, scatter) on any session/UI mutation, despite the correct selector pattern already existing in `packages/react/src/engineStore.ts:24-52`. `SelectionGizmo.tsx:402-461` compounds this by dispatching on every pointermove during drag.
54. **Command validation and rate-limit/auth middleware should ship as a library, not a per-game essay.** [K] host-side helpers (rate limit, session auth, schema-validate `CommandDefinition` input, reject unknown commands) belong in `@jgengine/ws`, reusable — currently only documented as a responsibility, not implemented.
55. **Dev/save endpoints have no documented production-off guarantee.** [K] editor save + dev save endpoints are powerful local tools; nothing in `create` scaffolding or doctor enforces stripping them from production builds.

## What survives — don't refactor away while cleaning up

Consensus across all three: zero-dep `core` + one-way layering, structural adapters (sql pool shape, transport pipe), catalogs-as-data + commands-as-verbs, deterministic/injectable RNG where actually used, dense pure-function test coverage next to modules, `useSyncExternalStore` pattern in `packages/react/src/engineStore.ts`, `guard.ts` hard-timeout-over-silent-hang policy, Credits/Kenney-ban discipline (enforcement gap aside), capability-index/orphan-gate *concept* (baseline just needs to shrink instead of grow).

## Suggested execution order

1. Phase 0 as one security-focused change set (items 1-10), before anything else ships.
2. Item 11 (`gameContext.ts` split) before Phase 1 continues — it's the root cause of 14, 25, 29, 37.
3. Item 13 (real PR CI) as infrastructure — do this early so every subsequent fix is actually gated.
4. Items 16-35 (P1) in parallel workstreams once P0 is closed: persistence/replication correctness, editor validation, package/version integrity, content-governance gating.
5. P2/P3 opportunistically, folded into whatever PR is already touching that file.
