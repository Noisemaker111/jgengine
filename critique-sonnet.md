# JGengine Critique (Sonnet)

Full-repo audit, organized by severity/phase. Every item has file:line and a one-line reason. Ignore `critique-grok.md` — unrelated, separate pass.

---

## Phase 0 — Ship-blocking (scale, data-loss, security)

- [ ] **Single global queue serializes every room on a host process.** [`packages/ws/src/host.ts:138-143`](packages/ws/src/host.ts:138) — one `Promise` chain (`enqueue`) gates `joinServer`/`runCommand`/`tickOnce`/`flushAll` for *all* rooms. A slow persistence write for room A blocks command processing for every unrelated room on the same host. Caps a process to serial throughput across the whole player base, not per-room concurrency — directly contradicts "scale is a default, not a feature."
- [ ] **Broadcast fan-out is O(total connections), not O(room size).** [`packages/ws/src/hostRouter.ts:148-155,173-180,197-204`](packages/ws/src/hostRouter.ts:148) — `broadcastPresence`/`broadcastChat`/`broadcastVoice` iterate the global `connections` Set for every message, regardless of which room it's for. Compounds with the missing rate limits below.
- [ ] **No rate limiting on pose/runCommand/join/browse/voice — only chat is throttled.** [`packages/ws/src/hostRouter.ts:158-160`](packages/ws/src/hostRouter.ts:158) builds a `chatLimiter`; `pose` at [`hostRouter.ts:342-371`](packages/ws/src/hostRouter.ts:342) has none. A single client can flood pose messages at network speed, each triggering a global-Set broadcast (see above) — real DoS ceiling well below "a handful of concurrent players."
- [ ] **Unchecked cast of client input straight into game logic.** [`packages/node/src/worldHost.ts:80`](packages/node/src/worldHost.ts:80) — `entry.session.input(userId, input as InputFrame)`, no runtime shape validation on attacker-controlled data.
- [ ] **Hosted-world sessions have no persistence by default — a restart loses everything, not just a flush window.** [`packages/node/src/worldServer.ts:50`](packages/node/src/worldServer.ts:50) calls `createHostedWorldSession(...)` without a `store`, defaulting to in-memory (`hostedWorldSession.ts:83`). `WorldGameServerOptions` exposes no plug point for a durable store.
- [ ] **No atomicity for memory/file persistence flush path.** [`packages/ws/src/host.ts:179-202`](packages/ws/src/host.ts:179) awaits `saveProfile`/`saveChunks`/`deleteChunks`/`saveServer` sequentially, no rollback. Only `sql`'s `savePlan` wraps this in a real transaction ([`packages/sql/src/sqlPersistence.ts:135-168`](packages/sql/src/sqlPersistence.ts:135)) — everyone else risks torn writes on crash.
- [ ] **No SIGINT/SIGTERM shutdown hook anywhere in `node`/`ws`.** Confirmed via repo-wide grep — nothing calls `host.stop()` (which flushes dirty servers) on process exit. Every deploy that doesn't wire this itself loses in-flight state on redeploy/OOM-kill.
- [ ] **The "never Kenney" rule is already broken, in a shipped game, silently.** [`Games/wreckway/src/game/assets.ts:7`](Games/wreckway/src/game/assets.ts:7) declares `sources: ["kenney-racing", "kenney-survival", "kenney-city-roads"]`; [`Games/wreckway/src/game/models.ts:30,60,124,...`](Games/wreckway/src/game/models.ts:30) references ~20 `kenney-*` ids. No such provider exists in `packages/assets/src/sources/index.ts`, so every reference silently no-ops to a primitive-shape fallback — this is simultaneously a hard rule violation and a live bug (wreckway's vehicles/props never render as real models). Nothing catches it: `packages/assets/src/verify.ts:21-61` never checks provider/license value, and no CI job runs asset verification at all.
- [ ] **Heavy CI checks don't gate PRs.** [`.github/workflows/ci.yml:31-49`](.github/workflows/ci.yml:31) — `checks` (test/check-types/build) and `web-build`/`smoke` run `if: github.event_name == 'push'` only; on `pull_request` only the ~30s `quick` job runs, and the final gate treats skipped jobs as passing. Broken types/tests/build can merge through a PR review and only surface after landing on `main`.
- [ ] **Tauri desktop disables CSP entirely, and wraps arbitrary user-supplied URLs in that webview.** [`apps/desktop/src-tauri/tauri.conf.json:21-23`](apps/desktop/src-tauri/tauri.conf.json:21) sets `"csp": null`; [`packages/jgengine/src/desktop.ts:141-153,427-430`](packages/jgengine/src/desktop.ts:141) lets `--url` load any HTTPS URL into that same webview — an attacker-controlled or compromised page gets a fully unrestricted native webview.

## Phase 1 — Architecture debt (the exact anti-pattern CLAUDE.md calls out)

- [ ] **`gameContext.ts` is the god-object CLAUDE.md warns against.** [`packages/core/src/runtime/gameContext.ts:506`](packages/core/src/runtime/gameContext.ts:506) — 1,763 lines, 59 imports across ~30 subsystems, hand-wires every optional feature with `features.x ? create... : undefined` branches (e.g. [lines 1327-1344](packages/core/src/runtime/gameContext.ts:1327)). Every new opt-in subsystem means editing this one function — "an engine file edited for every new variant is the smell," verbatim.
- [ ] **Same pattern in the editor domain.** [`packages/core/src/editor/commands.ts:250-717`](packages/core/src/editor/commands.ts:250) — `applyMutating`, 467 lines, 43 `case` labels, one branch per `EditorCommand` variant.
- [ ] **Two fully dead subsystems still ship in `dist`.** [`packages/core/src/store/engineStore.ts`](packages/core/src/store/engineStore.ts) (`EngineStore` class) and [`store/features.ts`](packages/core/src/store/features.ts) (`composeStateFeatures`) — zero imports anywhere in `src`, no tests, superseded by `defineStore`/`observableKeyedStore`. Delete or justify.
- [ ] **Real circular domain coupling between `world` and `editor`.** [`packages/core/src/world/scatterRegion.ts:1`](packages/core/src/world/scatterRegion.ts:1) and [`world/vegetation.ts:1`](packages/core/src/world/vegetation.ts:1) import from `../editor/types`, while [`editor/commands.ts:12`](packages/core/src/editor/commands.ts:12)/`document.ts`/`types.ts` import from `../world/terraform`. Type-only so it compiles, but it's genuine bidirectional domain entanglement. Same pattern between `combat/death.ts` ↔ `game/events.ts` and `render/lookPreset.ts` ↔ `game/playableGame.ts`.
- [ ] **`@internal` is stamped on ~18% of all exports, including public API front doors.** 570 occurrences vs 3,151 `export`s. [`packages/core/src/ai/spawnPoint.ts:11`](packages/core/src/ai/spawnPoint.ts:11) (`selectSpawnPoint`) is tagged `@internal` yet re-exported as flagship public API at [`authoring.ts:49`](packages/core/src/authoring.ts:49). A tag meant to mark true internals is meaningless at this hit rate.
- [ ] **API leaks internal mechanics the design principles explicitly ban.** [`packages/core/src/ai/spawnDirector.ts:60-67`](packages/core/src/ai/spawnDirector.ts:60) docs `SpawnPointSelectionOptions` as hiding "weighting internals," then exposes `biasStrength?: number` directly, fed straight into `Math.pow` as an exponent at [lines 265,274](packages/core/src/ai/spawnDirector.ts:265). `HeatState.rng: number` ([`ai/heatSystem.ts:36`](packages/core/src/ai/heatSystem.ts:36)) and `SpawnDirectorState.rng: number` ([`spawnDirector.ts:39`](packages/core/src/ai/spawnDirector.ts:39)) leak raw PRNG state into public return types — contrast with the correct injected-RNG pattern in [`random/pick.ts:6,12`](packages/core/src/random/pick.ts:6).
- [ ] **The one truly untrusted input boundary (editor RPC/CLI) is also the least type-safe path.** [`packages/editor/src/mcp/cli.ts:68`](packages/editor/src/mcp/cli.ts:68) and [`stdioServer.ts:147`](packages/editor/src/mcp/stdioServer.ts:147) cast dynamically-imported, unvalidated game-authored `editorLayers` modules with `as never` right before feeding them into `createEditorHost`. [`loadGameLayers.ts:2-11`](packages/editor/src/mcp/loadGameLayers.ts:2) returns `unknown` with zero schema validation — a malformed game module gets no compile- or run-time check before hitting a live session.
- [ ] **`EditorDocument` is versioned (`version: 1`) with no migration path**, despite `migrateTerrainSnapshot` existing for the nested `terrain` field. [`packages/core/src/editor/types.ts:104-116`](packages/core/src/editor/types.ts:104), [`editor/document.ts:1,91`](packages/core/src/editor/document.ts:1). The next scene-format bump has no defined upgrade story.

## Phase 2 — Testing gaps

- [ ] **Core: several real-logic subsystems have zero direct tests.** `runtime/transport.ts`, `multiplayer/presenceContract.ts`, `devtools/tunableSchema.ts` (605 lines), `world/buildingGenerator.ts`, `render/postProcessing.ts`, `audio/synth.ts`. Worst: [`runtime/gameRuntime.ts:72-172`](packages/core/src/runtime/gameRuntime.ts:72) — hosted-server join/tick/dirty-tracking logic, reached by only 1 test transitively, no direct unit test of join/tick/dirty semantics.
- [ ] **Editor GUI panels: 0 of 14 `.tsx` files tested.** `AssetBrowser`, `OutlinerPanel`, `SelectionGizmo`, `TerrainSculpt`, `PrefabsPanel`, `CollectionsPanel`, `EditorApp` itself — all untested, despite the editor being the sole intended path for all scene authoring. Editor overall: 5/28 files (~18%).
- [ ] **Multiplayer race conditions are essentially untested end-to-end.** [`packages/ws/src/hostRouter.test.ts:328`](packages/ws/src/hostRouter.test.ts:328) tests same-socket message serialization, but nothing tests multiple concurrent clients racing writes to the same room, out-of-order message delivery, or a client reconnecting mid-session at the router level (reconnect is only tested from the client backend's side, [`createWsBackend.test.ts:106`](packages/ws/src/createWsBackend.test.ts:106)).
- [ ] **`convex` package is the thinnest backend layer by coverage.** `convexChatTransport.ts`, `convexPresenceTransport.ts`, `createConvexBackend.ts`, `createConvexGameTransport.ts`, `resolveConvexMultiplayer.ts` — no dedicated test file.
- [ ] **Two games ship with essentially no regression coverage.** `spire-cards` and `studio-showcase` — 1 test file each against 16 and 12 source files respectively.

## Phase 3 — Editor/React runtime issues

- [ ] **`EditorWorldOverlay` re-renders its entire subtree on any session/ui mutation.** [`packages/editor/src/EditorApp.tsx:117-120`](packages/editor/src/EditorApp.tsx:117) uses three separate `useState(0)`+`setTick` "force update" subscriptions instead of `useSyncExternalStore` with per-slice selectors — the correct pattern the team already uses in [`packages/react/src/engineStore.ts:24-52`](packages/react/src/engineStore.ts:24). Every session/ui change re-renders `PerfProbe`, `EditorCameraDriver`, `MaterialDropZone`, `TerrainSculpt`, `ScatterPreview`, `EditorLayerOverlays`, `SelectionGizmo` together.
- [ ] **`SelectionGizmo` dispatches on every pointermove during a drag.** [`packages/editor/src/SelectionGizmo.tsx:402,413-461`](packages/editor/src/SelectionGizmo.tsx:402) — dragging a gizmo re-renders the overlay tree at pointer-event frequency, not on drag-end/throttled.
- [ ] **One hardcoded, non-headless overlay in the editor.** [`EditorModeChip`, `EditorApp.tsx:152-166`](packages/editor/src/EditorApp.tsx:152) — fixed `absolute inset-x-0 top-2` pill with baked-in copy/emoji. Low severity since it's editor chrome not game UI, but it's the one exception to "chrome is composable, never imposed" found in scope.

## Phase 4 — Content governance (games violating the engine's own rules)

- [ ] **7 of 10 games hardcode placement/geometry the "author in editor" rule bans — and nothing gates it.** Only `tower-guard` and `studio-showcase` have an `editor.scene.json` consumed via `<AuthoredScene>`. `scripts/check-game-shape.ts` checks file layout only, no content-level check exists.
  - [`Games/the-robots/src/game/world/level.ts:291-372`](Games/the-robots/src/game/world/level.ts:291) — worst offender: hand-written `switch(zone.id)` with ~90 magic-number placement calls, plus a hand-rolled road wall/ramp mesh generator ([lines 122-211](Games/the-robots/src/game/world/level.ts:122)) — literally the "per-segment path mesh" the rule names.
  - [`Games/vice-isle/src/game/world/districts.ts:12-84`](Games/vice-isle/src/game/world/districts.ts:12) — `DISTRICTS`, `ROADS` (9 segments), `RACE_CHECKPOINTS` (10 literal coordinates) feeding waypoints in `handroll.ts:390`.
  - [`Games/claudecraft/src/game/arena/catalog.ts:25-56`](Games/claudecraft/src/game/arena/catalog.ts:25) — literal arena pillar/spawn coordinate arrays.
  - [`Games/loopline/src/game/world/setup.ts:14-73`](Games/loopline/src/game/world/setup.ts:14) — an entire starter park (rides, 10-point track loop, 19-point paths, 16 trees) hand-authored as coordinate literals.
- [ ] **`CREDITS.md` doesn't credit KayKit or Quaternius**, despite the last 5+ PRs (#833, #831, #829, #827, #803, #802, #800) re-homing flagship game art onto exactly those packs, and both having first-class source modules in `packages/assets/src/sources/`. - [ ] **`packages/assets/README.md` documents a `src/sources/kenney.ts` and `KENNEY_PACKS` that don't exist**, including a live `pull kenney-food` example — stale docs pointing at a source the repo explicitly bans.

## Phase 5 — DX / tooling / monorepo hygiene

- [ ] **`jgengine create` prints corrupted mojibake to every user's terminal.** [`packages/jgengine/src/create.ts:128,145,152,158`](packages/jgengine/src/create.ts:128) — source bytes are double-encoded UTF-8 garbage (`ΓåÆ`, `ΓÇª`, `ΓÇö` in place of `→`/`…`/`—`). Not a terminal rendering issue — the file itself is corrupted.
- [ ] **CLI hard-requires `bun` on PATH for the editor subcommand, no npm/pnpm fallback**, despite `--pm` support existing elsewhere in `create`. [`packages/jgengine/src/create.ts:37`](packages/jgengine/src/create.ts:37).
- [ ] **`apps/web`'s Vercel deploy trigger is far broader than its actual dependency.** [`apps/web/vercel.json:2`](apps/web/vercel.json:2) diffs `HEAD^..HEAD` against `apps/web apps/dev packages Games .claude/skills` wholesale, but `apps/web/package.json` shows the site only depends on `@jgengine/github`. An unrelated tweak to any single `Games/*` playtest or any unrelated package forces a full site rebuild/redeploy on every tracked push.
- [ ] **All 10 games + both non-web apps duplicate an identical `devDependencies` block** (`@tailwindcss/vite`, `@types/react`, `@types/three`, `vite`, etc.) with no shared base config — a dozen copies to keep in sync by hand.
- [ ] **Unexplained major-version split: `apps/web` pins `vite ^8.1.3`/`@vitejs/plugin-react ^6.0.3` while every game and other app pins `vite ^6.2.2`/`^4.3.4`.**
